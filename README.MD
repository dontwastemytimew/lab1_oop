# Lab 1: Refactoring & Object-Oriented Design (Notes Manager)

**Structured Notes Manager** — це застосунок на C++ для керування нотатками з підтримкою динамічних схем даних.

---

## Архітектура та Рефакторинг

Проєкт було перебудовано з монолітної структури на модульну архітектуру:

### 1. Розділення на компоненти
Проєкт розділено на незалежні модулі:
* **CoreLib:** Містить бізнес-логіку, моделі даних (`Note`, `Schema`) та сервіси. Не залежить від графічного інтерфейсу.
* **GUI:** Відповідає виключно за відображення та взаємодію з користувачем.

### 2. Патерни проєктування
* **Repository Pattern:** Клас `NoteRepository` інкапсулює логіку роботи з колекцією нотаток.
* **Strategy Pattern:** Інтерфейс `IStorageStrategy` дозволяє динамічно змінювати алгоритм серіалізації даних.
* **Service Layer:** Виділено спеціалізовані сервіси (`ExportService`, `StatisticsService`) для дотримання принципу SRP.

## Функціональні можливості

1.  **Динамічні схеми:** Створення власних типів нотаток.
2.  **Smart Search & Sort:** Фільтрація за текстом, тегами, датою.
3.  **Експорт:** Збереження нотаток у **PDF** або **JSON**.
4.  **Статистика:** Візуалізація активності користувача (графіки **QCustomPlot**).
5.  **UI/UX:** Темна тема, закріплення нотаток, підтримка зображень.
