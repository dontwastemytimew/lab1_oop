ГЛОСАРІЙ ПРЕДМЕТНОЇ ОБЛАСТІ ТА АРХІТЕКТУРИ
1. Основні сутності (Domain Entities)
Нотатка (Note) — основна одиниця інформації в системі, що містить заголовок, дату створення, набір тегів, зображення та колекцію динамічних полів.

Схема (Schema) — шаблон або структура, що визначає склад та типи полів (текст, число тощо) для конкретної категорії нотаток.

Тег (Tag) — ключове слово, що використовується для групування та швидкого пошуку нотаток незалежно від їхньої схеми.

Поле (Field) — елемент даних у складі нотатки, тип якого (текстовий або числовий) диктується обраною схемою.

2. Архітектурні компоненти (Technical Layers)
Репозиторій (NoteRepository) — компонент, що інкапсулює логіку доступу до даних у пам'яті. Він забезпечує інтерфейс для CRUD-операцій (створення, читання, оновлення, видалення) та приховує деталі збереження.

Стратегія збереження (IStorageStrategy) — абстрактний інтерфейс, що визначає контракт для серіалізації та десеріалізації даних. Дозволяє змінювати формат файлу (наприклад, з Nlohmann JSON на RapidJSON) без зміни логіки репозиторію.

Шар сервісів (Service Layer) — набір класів (ExportService, StatisticsService), які містять специфічну бізнес-логіку, що не належить безпосередньо до керування станом об'єктів.

Ін'єкція залежностей (Dependency Injection) — підхід, при якому конкретна реалізація стратегії передається в репозиторій ззовні (через конструктор), що зменшує зв'язність модулів.

3. Формати та серіалізація
JSON (JavaScript Object Notation) — текстовий формат обміну даними, що використовується в програмі як основний спосіб довготривалого зберігання.

Base64 — метод кодування бінарних даних (зображень) у текстовий рядок для їх безпечного зберігання всередині JSON-файлу.

PDF (Portable Document Format) — формат експорту нотаток, що забезпечує незмінність візуального представлення документа при друку або перегляді.
