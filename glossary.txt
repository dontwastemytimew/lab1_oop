ГЛОСАРІЙ ПРОЄКТУ "STRUCTURED NOTES MANAGER"
Цей документ містить опис ключових концепцій предметної області, архітектурних рішень та технологій, використаних у розробці системи.
1. Основні сутності (Domain Entities)
Нотатка (Note) — основна одиниця інформації в системі, що містить заголовок, дату створення, набір тегів, зображення та колекцію динамічних полів.

Схема (Schema) — шаблон або структура, що визначає склад та типи полів (текст, число тощо) для конкретної категорії нотаток.

Тег (Tag) — ключове слово, що використовується для групування та швидкого пошуку нотаток незалежно від їхньої схеми.

Поле (Field) — елемент даних у складі нотатки, тип якого (текстовий або числовий) диктується обраною схемою.

2. Архітектурні компоненти та ООП
Репозиторій (NoteRepository) — компонент, що інкапсулює логіку доступу до даних у пам'яті. Він забезпечує інтерфейс для CRUD-операцій та приховує деталі збереження.

Стратегія (Strategy Pattern) — патерн проєктування, реалізований через інтерфейс IStorageStrategy. Він дозволяє змінювати алгоритм збереження даних (JSON/RapidJSON) без модифікації коду репозиторію.

Шар сервісів (Service Layer) — набір класів (ExportService, StatisticsService), які містять специфічну бізнес-логіку (експорт в PDF, аналітика), що не належить до керування станом об'єктів.

Інкапсуляція (Encapsulation) — приховування внутрішньої реалізації класів (наприклад, полів m_notes у DataManager) та надання доступу до них лише через публічні методи.

Поліморфізм (Polymorphism) — здатність системи працювати з різними стратегіями збереження через спільний інтерфейс IStorageStrategy, не знаючи про конкретну реалізацію в момент виклику.

Ін'єкція залежностей (Dependency Injection) — підхід, при якому конкретна реалізація стратегії передається в репозиторій ззовні (через конструктор), що зменшує зв'язність модулів.

3. Технології та Бібліотеки
Qt 6 — основний фреймворк для розробки кросплатформного графічного інтерфейсу (GUI) та керування подіями.

nlohmann/json — сучасна C++ бібліотека для роботи з форматом JSON, що цінується за зручний синтаксис та легкість інтеграції.

RapidJSON — високопродуктивна бібліотека для парсингу та генерації JSON, використана в проєкті для оптимізації швидкості збереження великих обсягів даних.

QCustomPlot — зовнішня бібліотека для Qt, використана для візуалізації статистики використання програми у вигляді графіків та діаграм.

nanobench.h — легковаговий фреймворк для проведення мікро-бенчмаркінгу та точного вимірювання часу виконання функцій у наносекундах.

4. Формати та Обробка даних
JSON (JavaScript Object Notation) — текстовий формат обміну даними, що використовується як основний спосіб довготривалого збереження бази нотаток.

Base64 — метод кодування бінарних даних (зображень) у текстовий рядок для їх безпечного зберігання всередині JSON-файлу.

PDF (Portable Document Format) — формат експорту нотаток, що забезпечує стабільне візуальне представлення документа при друку.

Серіалізація (Serialization) — процес перетворення об'єктів програмного коду (нотаток, схем) у формат, придатний для збереження у файл (JSON).

5. Інструменти розробки (DevOps & Docs)
CMake — система автоматизації збирання проєкту, що керує компіляцією та лінкуванням бібліотек.

Doxygen — інструмент для автоматичної генерації документації з коментарів у вихідному коді програми.

GitHub Actions — сервіс автоматизації (CI/CD), що використовується для автоматичного збирання документації та деплою на GitHub Pages при кожному оновленні коду.

PlantUML — інструмент для створення UML-діаграм (класів, послідовності тощо) за допомогою декларативного текстового опису.
